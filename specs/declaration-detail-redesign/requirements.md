# Requirements: עיצוב מחדש דף פרטי הצהרה

## סקירה

עיצוב מחדש של דף פרטי ההצהרה (`/dashboard/declarations/[id]`) כדי לתעדף מידע חשוב למשרד (דדליינים, קנסות, רו"ח אחראי) על פני פעילות לקוח (כניסה לפורטל, העלאת מסמכים).

## בעיות נוכחיות

1. **היסטוריית סטטוסים ענקית** - תופסת יותר מדי מקום, כותרת כפולה
2. **אין תעדוף** - סטטוסי לקוח באותה חשיבות כמו סטטוסי משרד
3. **מידע מפוזר** - דדליינים, רו"ח, קנסות בכרטיסים נפרדים
4. **היסטוריה כפולה** - סטטוסים ותקשורת בנפרד

## דרישות

### היררכיית מידע

**קריטי (בולט מאוד):**
- קנסות והתראות
- דדליינים קרובים

**חשוב (Main content):**
- סטטוס משרד (reviewing, in_preparation, submitted)
- מסמכים
- רו"ח אחראי

**רגיל:**
- היסטוריה מאוחדת (סטטוסים + תקשורת)

**משני (Sidebar):**
- פורטל לקוח
- פרטי לקוח (ת.ז.)

### קריטריוני קבלה

- [ ] כרטיס סטטוס משרד מאוחד עם דדליינים + רו"ח + קנסות
- [ ] Timeline מאוחד (סטטוסים + תקשורת) עם "הצג עוד"
- [ ] הבחנה ויזואלית בין פעולות משרד (כחול) לפעולות לקוח (אפור)
- [ ] Sidebar מצומצם (רק פורטל + ת.ז.)
- [ ] הצגת מצב ריק עם הודעה מתאימה
- [ ] שמירה על תאימות RTL
- [ ] עובד ב-mobile
